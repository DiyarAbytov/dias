ФРОНТ ВЫЗЫВАЕТ — СЕЙЧАС 404

POST http://127.0.0.1:8000/api/orders/{id}/release/

Примеры:
  POST /api/orders/1/release/
  POST /api/orders/2/release/

Тело запроса (JSON):
  {
    "quantity": 12
  }

где quantity — количество выпускаемой продукции (число, шт).


ЧТО НУЖНО ОТ БЭКА

1. Реализовать endpoint:
   POST /api/orders/{order_id}/release/
   — зарегистрировать в роутере (например, в urlpatterns для orders).
   — если уже есть view (например ProductionReleaseView), привязать его к маршруту вида orders/<id>/release/.

2. При успешном выпуске:
   — списание со складов (по вашей логике),
   — создание партии (batch), чтобы она появлялась в GET /api/batches/ и во вкладке ОТК «Ожидают».

3. Ответ:
   — 200 OK и тело (например, созданная партия или обновлённый заказ) — формат на усмотрение бэка;
   — при ошибке (заказ не найден, нельзя выпустить и т.д.) — 4xx с текстом ошибки.


ФРОНТ ОЖИДАЕТ

— Успех: статус 2xx (после этого фронт обновляет список заказов и список партий).
— Ошибка: не 2xx (фронт покажет сообщение из ответа).

Итог: нужно, чтобы POST /api/orders/{order_id}/release/ существовал и был зарегистрирован в API (например, в config/api_urls.py или в роутере приложения orders).
